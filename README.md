🚗 자동차 경주 게임 (Racing Car Game)
MVC 패턴을 기반으로 사용자에게 자동차 이름과 시도 횟수를 입력받아 경주를 진행하고, 최종 우승자를 판별하는 콘솔 게임입니다.

🎯 핵심 기능
이름 입력 및 검증: 경주할 자동차 이름들을 쉼표(,) 기준으로 입력받습니다. 각 이름은 1자 이상 5자 이하여야 하며, 공백이나 빈 문자열은 허용하지 않습니다.

경주 진행: 시도 횟수만큼 경주를 진행하며, 각 턴마다 0~9 사이의 무작위 값을 뽑아 4 이상이면 전진, 3 이하면 정지합니다.

결과 출력: 매 턴마다 자동차의 이름과 현재 이동 거리를 출력하고, 게임이 끝나면 최종 우승자(단독 또는 공동)를 안내합니다.

📂 프로젝트 구조 (MVC 패턴 완벽 분리)
객체 지향적인 설계(단일 책임 원칙)와 테스트 용이성을 위해 계층의 역할을 명확히 분리했습니다.

model: 핵심 비즈니스 로직과 데이터 상태를 관리하며, View에 전혀 의존하지 않습니다.

view: 순수하게 콘솔 입출력만을 담당하며, 데이터 가공(Parsing) 책임을 가지지 않습니다.

controller: View로부터 원시 입력값(Raw String)을 받아 검증 및 가공하고, Model과 연결하여 게임의 전반적인 흐름을 제어합니다.

Application: 프로그램의 진입점(main)으로서, Controller를 생성하고 실행하는 역할만 담당합니다.

🛠️ 기능 구현 목록
1. Controller (controller)
   [x] RacingController: 전체 게임 로직의 흐름을 지휘한다.

[x] View에서 전달받은 원시 문자열에 대한 1차 예외(null, 빈 값)를 입구컷으로 검증한다.

[x] Stream을 활용해 문자열 분리(split)와 동시에 앞뒤 공백(trim())을 깔끔하게 제거한다.

[x] 시도 횟수를 정수형(int)으로 파싱하며, 문자나 0 이하의 숫자가 들어오면 예외를 발생시킨다.

2. Model (model) - 핵심 비즈니스 로직
   [x] Car: 자동차의 이름과 위치 상태를 관리한다.

[x] 이름이 5자를 초과할 경우, 또는 비어있거나 공백일 경우 각각 명확한 원인을 담은 IllegalArgumentException을 발생시킨다.

[x] 전달받은 숫자가 4 이상일 경우 위치를 1 증가시킨다.

[x] Cars (일급 컬렉션): 자동차 명단(List<Car>)을 래핑하여 관리한다.

[x] 생성 시 참가자가 2명 미만이거나 이름이 중복되면 예외를 발생시킨다.

[x] 모든 자동차에게 일괄 이동 명령을 내린다.

[x] 현재 가장 멀리 이동한 위치를 계산해 우승자 명단을 반환한다.

[x] NumberGenerator & RandomNumberGenerator: 난수 생성 로직을 인터페이스로 추상화하여 전략 패턴(Strategy Pattern) 기반의 테스트 가능한 구조를 만든다.

[x] RacingGame: 현재 게임의 턴을 관리하고, 게임 종료 여부를 판단한다.

3. View (view) - UI 로직
   [x] InputView: Scanner를 활용하여 사용자 입력을 통짜 문자열(String) 형태로 그대로 반환한다. (파싱 로직 제거)

[x] ResultView: 콘솔 출력을 전담한다.

[x] 매 턴마다 각 자동차의 이동 결과를 이름 : --- 형식으로 출력한다.

[x] 최종 우승자를 쉼표로 구분하여 출력한다.
