# 🚗 자동차 경주 게임 (Racing Car Game)

MVC 패턴을 기반으로 사용자에게 자동차 이름과 시도 횟수를 입력받아 경주를 진행하고, 최종 우승자를 판별하는 콘솔 게임입니다.

## 🎯 핵심 기능
- **이름 입력 및 검증**: 경주할 자동차 이름들을 쉼표(,) 기준으로 입력받으며, 각 이름은 **5자 이하**여야 합니다.
- **경주 진행**: 시도 횟수만큼 경주를 진행하며, 각 턴마다 0~9 사이의 무작위 값을 뽑아 **4 이상이면 전진, 3 이하면 정지**합니다.
- **결과 출력**: 매 턴마다 자동차의 이름과 현재 이동 거리를 출력하고, 게임이 끝나면 최종 우승자(단독 또는 공동)를 안내합니다.

---

## 📂 프로젝트 구조 (MVC 패턴)

객체 지향적인 설계와 테스트 용이성을 위해 도메인 로직과 UI 로직을 완벽히 분리했습니다.
- **Model**: 핵심 비즈니스 로직을 담당하며, View에 전혀 의존하지 않습니다.
- **View**: 콘솔 입출력을 담당합니다.
- **Controller (`Application`)**: View와 Domain을 연결하여 전체 애플리케이션의 흐름을 제어합니다.

---

## 🛠️ 기능 구현 목록

### 1. Model (핵심 비즈니스 로직)
- [x] **`Car`**: 자동차의 이름과 위치 상태를 관리한다.
    - [x] 이름이 5자를 초과하거나 비어있을 경우 `IllegalArgumentException`을 발생시킨다.
    - [x] 전달받은 숫자가 4 이상일 경우 위치를 1 증가시킨다.
- [x] **`Cars` (일급 컬렉션)**: 자동차 명단(`List<Car>`)을 감싸서 관리한다.
    - [x] 모든 자동차에게 이동 명령을 내린다.
    - [x] 현재 가장 멀리 이동한 최대 위치를 구하고 우승자 명단을 반환한다.
- [x] **`NumberGenerator` & `RandomNumberGenerator`**: 난수 생성 로직을 인터페이스로 추상화하여 **테스트 가능한 구조**를 만든다.
- [x] **`RacingGame`**: 현재 게임의 턴을 관리하고, 게임 종료 여부를 확인한다.

### 2. View (UI 로직)
- [x] **`InputView`**: `Scanner`를 활용하여 사용자 입력을 처리한다.
    - [x] 자동차 이름 목록을 입력받아 쉼표(,)로 분리한다.
    - [x] 시도할 회수를 정수로 입력받는다.
- [x] **`ResultView`**: 콘솔 출력을 전담한다.
    - [x] 매 턴마다 각 자동차의 이동 결과를 `이름 : ---` 형식으로 출력한다.
    - [x] 최종 우승자를 쉼표로 구분하여 출력한다.

### 3. Controller
- [x] **`Application`**: 입력 ➡️ 게임 객체 생성 ➡️ 턴 반복 진행 ➡️ 최종 우승자 출력의 전체 흐름을 제어한다.
